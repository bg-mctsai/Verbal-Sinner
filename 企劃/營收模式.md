《嘴砲王者》營收模式與 AI 成本優化方案

一、 面臨的挑戰：AI 成本分析

主要支出： 1. Gemini 2.5 API 呼叫： 每一回合的語意分析與 JSON 生成皆需計費。 2. TTS 語音生成： 語音合成的成本通常高於單純的文本生成。 3. 動態資源生成： 若在遊戲運行中即時產生圖片，成本極高且不穩定。

二、 營收模式 (Revenue Streams)

為了平衡支出，我們將採取「混合型營收模型」，在不破壞遊戲體驗的前提下變現。

1. 能量系統 (Energy/Heart System) —— 限制高頻損耗

機制： 每一回合「嘴砲」消耗 1 點能量。

變現： 能量隨時間恢復，或玩家可觀看激勵影片（Rewarded Video Ads）立即補滿。

目的： 確保每一筆 AI 呼叫都有相應的廣告收入支撐，避免玩家無限制刷對話導致 API 支出暴增。

2. 高級語音包與美學解鎖 (Premium Content)

機制： 基礎版使用預設語音。

變現： * 購買「特殊配音包」（更換 TTS 聲音參數，如：病嬌聲、霸總聲）。

購買「典藏場景圖」（更換高品質、具動態效果的背景）。

3. 道具系統 (Power-ups)

變現：

往事投影機：直接標示 NPC 的弱點標籤，減少無效嘗試。

沈默藥水：下一回合對手不會反擊玩家，減少玩家 EQ 的損耗。

三、 AI 成本優化策略 (Cost Optimization)

這部分是技術開發的關鍵，目的是減少不必要的 API 浪費。

1. 靜態資源預製化 (Asset Pre-rendering) —— 【核心優化】

策略： 將動態生成轉為靜態讀取。

做法： * 關卡背景：在開發階段使用 AI 生成 100 關卡的背景圖，篩選後存入 Firebase Storage。

角色立繪：預先繪製好 NPC 的「喜、怒、哀、樂」四種狀態圖，根據 AI 回傳的情緒標籤直接切換。

效果： 將圖片生成成本從「每次遊玩」降至「一次性開發成本（趨近於零）」。

2. 語義快取機制 (Semantic Caching)

策略： 使用 Firestore 建立「語句指紋庫」。

做法： 若玩家輸入的文字與歷史庫中高度重複（例如常見的挑釁辭彙），直接返回快取好的 AI 結果，無需重複調用 API。

3. 語音分級處理 (Voice Layering)

策略： 僅核心對話使用語音。

做法： 雜魚關卡僅顯示文字；只有各章節的 10/10 最終關卡 或重要劇情點才觸發語音生成。

4. 降低模型精度 (Model Switching)

做法： 一般劇情引導使用更便宜的小模型，只有在需要深度判斷「是否扎心」的邏輯博弈時才呼叫 gemini-2.5-flash-preview。

四、 營收平衡試算 (Estimations)

| 項目              | 預估單次成本 (USD) | 補償手段      | 損益平衡點                   |
| ----------------- | ------------------ | ------------- | ---------------------------- |
| 一回合嘴砲 (Text) | ~$0.00x            | 激勵廣告收入  | 每 10 回合看一次廣告即可盈利 |
| 語音生成 (TTS)    | ~$0.0x             | 道具內購/訂閱 | 需有 5% 付費轉化率           |
| 圖片背景 (Static) | $0.00(預製)        | 無需補償      | 純利潤空間                   |

五、 結論：以「效率」為核心的運營

透過預先製作場景圖與角色立繪，我們成功將最昂貴的圖片生成成本消除。目前的成本結構將集中在「即時語意對話」上。透過能量系統與廣告的結合，能確保遊戲在擴大用戶規模的同時，依然保持健康的盈利能力。

「預製視覺，動態靈魂；讓每一分預算都花在最具感染力的對話上。」
